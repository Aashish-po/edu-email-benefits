## 🔐 Verification Methods

### Method 1: `.edu` Email Auto-Verification ⭐ **FASTEST**

**How it works:**
1. Enter your `.edu` email at signup
2. Service checks your domain against EDUCAUSE registry
3. **Instant access** — no additional steps

**Which services use this:**
- JetBrains, Figma, Notion, Lucidchart, Microsoft Office 365, Miro, Stack Overflow Teams, ResearchGate, and 20+ others

**Pro tip:** Register for everything using your `.edu` email — it's the fastest path to verification.

---

### Method 2: SheerID Verification ⚡ **FAST**

**How it works:**
1. Click "Verify with SheerID"
2. Enter your `.edu` email OR upload enrollment document
3. Usually instant (sometimes takes 24–48 hours)

**Which services use this:**
- Spotify Premium Student, Adobe Creative Cloud, Perplexity Pro, DataCamp, Julius AI, and others

**Pro tip:** SheerID is third-party, so the same verification works across multiple services.

---

### Method 3: GitHub Education Pack 📦 **MOST VALUABLE**

**How it works:**
1. Apply at [education.github.com/pack](https://education.github.com/pack)
2. Verify with `.edu` email or upload student ID photo
3. Approval takes 1–7 days

**Benefits:** Unlocks 30+ deals instantly, including GitHub Copilot Pro, $200 DigitalOcean credit, Namecheap free domain, and more.

**Pro tip:** Apply immediately — it covers many popular tools and is the single most valuable student verification.

---

### Method 4: Institutional Access 🏫

**How it works:**
Some benefits are **automatically available** through your school's subscription:
- Microsoft Office 365 (most universities)
- Zoom Pro (most universities)
- LinkedIn Learning (many universities)
- Canvas, Blackboard integrations
- Adobe Creative Cloud (some universities)

**Check with your IT department** — you might have access already without any application.

## 📌 Best Practices

### 1. **Create a `.edu` email forwarding rule**
Most student benefits verify against your `.edu` address. Set up email forwarding to your personal Gmail/Outlook so you don't miss notifications:

```
your-name@university.edu → yourpersonal@gmail.com
```

### 2. **Use a password manager**
You'll create accounts across 30–50 services. Use [Bitwarden](https://bitwarden.com/), [1Password](https://1password.com/), or [KeePass](https://keepass.info/) to track credentials.

### 3. **Set yearly re-verification reminders**
Most benefits require you to reverify student status annually. Add these to your calendar:
- GitHub Education Pack
- JetBrains IDEs
- Spotify Premium Student
- Microsoft Office 365
- Adobe Creative Cloud

### 4. **Stack your benefits strategically**
Combine free tools to create a complete workflow:

| Use Case | Tools | Total Cost |
|---|---|---|
| **Web Development** | JetBrains WebStorm + GitHub Copilot + MongoDB Atlas + DigitalOcean | **FREE** |
| **UI/UX Design** | Figma + Autodesk (3D) + Adobe (video) | **~$20/month** |
| **Data Science** | VS Code + Google Colab + DataCamp | **~$12.50/month** |
| **Game Dev** | Unity Pro + Unreal Engine | **FREE** |
| **Content Creation** | Adobe Creative Cloud + Canva + Spotify | **~$30/month** |

### 5. **Keep your student status current**
Once you graduate:
- Most benefits expire automatically
- Some allow grace periods (30–90 days)
- A few let you retain access if paid during your last semester

Found a benefit that's missing or outdated? We'd love your contribution!

### Adding a new benefit

1. **Verify the benefit is current** — test it yourself with a `.edu` email
2. **Open `README.md`** and find the appropriate category
3. **Add an entry in this format:**

```markdown
| [Service Name](official-link) | 🟢 **FREE** | One-line description of the benefit | `.edu` email auto-verify | [→ Claim](link) |
```

4. **Update the count** in the main heading

### Adding a new category

If you've found multiple related benefits not covered by existing categories:

1. Create a new section heading
2. Add it to the Table of Contents
3. Format exactly like other sections
4. Submit a PR with your new category

### Pull request guidelines

```bash
# 1. Fork this repo
git clone https://github.com/yourusername/edu-email-benefits.git
cd edu-email-benefits

# 2. Create a feature branch
git checkout -b add/service-name

# 3. Edit README.md
# - Add your benefit to the correct section
# - Update the count in the main header
# - Keep formatting consistent

# 4. Commit with a clear message
git commit -m "add: ServiceName student benefit with .edu verification"

# 5. Push and open a PR
git push origin add/service-name
```

**PR Checklist:**
- [ ] Benefit is verified to be active and current

- [ ] Verification method is accurate

1. **Claim everything at signup** — you won't use every tool, but they're free. Cancel later if needed.
